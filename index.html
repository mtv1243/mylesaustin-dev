Stack:

Abstract

Online Cribbage is a project that lets you create a shareable link to a unique, self contained game of cribbage you can play with a friend. You can read in detail about how it was created in this series of medium articles.

Briefly, the Node Express server creates a unique, sharable url to a private game. This approach uses Node's built-in fs module to read the contents of a template file and add in a unique gamekey that is used to route HTTP requests to that game, as well as create a new dedicated realtime database node. Doing it this way lets Node do all the work, rather than outsourcing dynamic URL creation to a third party package or library. Read in detail about the server's code here.

The client-side JavaScript handles all of the logic for the actual card game data, such as each player's cards, their scores, etc. Read about the front-end code here. The data is all stored in a Firebase Realtime Database node unique to that game. The guiding principle in this implementation is that the database should control the DOM. This will ensure that if you see a “card” element in the DOM, that card represents data from the database available to both players, and is not just a local DOM element on your machine that your opponent will not see.

If the app were to get larger, and have more users with more permanent data, such as usernames or a history of wins and losses on the platform, there would likely be a need to have a more robust data storage solution. Because of the shallow structure of the data, and because each game is totally unique, a NoSQL database such as MongoDB would probably be a good candidate.
